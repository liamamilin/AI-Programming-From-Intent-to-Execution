22  Multi-Agent Systems

22.1 From Single Agent to Systems of Agents

So far, we have considered systems where:

  • A single agent receives input

  • Processes it

  • Produces output

This works for simple or moderately complex tasks.

However, as systems grow:

  • Tasks become multi-step

  • Knowledge becomes specialized

  • Decisions become more complex

A single agent becomes insufficient.

This leads to:

Multi-Agent Systems

22.2 What is a Multi-Agent System?

A Multi-Agent System is a system where:

Multiple agents collaborate to achieve a shared goal

Each agent:

  • Has a defined role

  • Performs specific tasks

  • Interacts with other agents

22.2.1 Key Idea

Instead of one agent doing everything:

  • Work is distributed

  • Responsibilities are separated

22.3 Why Multiple Agents?

There are several reasons to move beyond a single agent.

22.3.1 1. Specialization

Different tasks require different capabilities.

  • Retrieval

  • Reasoning

  • Planning

  • Execution

22.3.2 2. Complexity Management

Breaking a problem into parts:

  • Reduces cognitive load

  • Improves clarity

22.3.3 3. Modularity

Agents can be:

  • Reused

  • Replaced

  • Extended

22.3.4 4. Scalability

Systems can grow by:

  • Adding more agents

  • Expanding capabilities

Multi-agent systems scale by composition.

22.4 Roles and Responsibilities

In a multi-agent system, each agent must have:

  • A clear role

  • Defined responsibilities

22.4.1 Example Roles

22.4.1.1 Planner

  • Breaks down tasks

  • Defines steps

22.4.1.2 Executor

  • Performs actions

  • Calls tools

22.4.1.3 Evaluator

  • Checks results

  • Provides feedback

22.4.1.4 Retriever

  • Fetches external information

Each agent should do one thing well.

22.5 Division of Labor

Division of labor is central to multi-agent systems.

22.5.1 Principles

22.5.2 1. Separation of Concerns

  • Each agent handles a distinct function

22.5.3 2. Clear Boundaries

  • Inputs and outputs are well-defined

22.5.4 3. Minimal Overlap

  • Avoid duplication of responsibilities

22.5.5 Example Flow

User Query
→ Planner Agent
→ Executor Agent
→ Evaluator Agent
→ Final Output

Clear division improves reliability.

22.6 Communication Between Agents

Agents must communicate to coordinate actions.

This requires:

Defined communication protocols

22.6.1 What is Communicated?

  • Tasks

  • Intermediate results

  • Feedback

22.6.2 Forms of Communication

22.6.2.1 Structured Messages

  • JSON

  • Key-value pairs

22.6.2.2 Natural Language

  • Flexible

  • Less reliable

22.6.2.3 Hybrid

  • Structured + descriptive

Structured communication improves consistency.

22.7 Communication Patterns

22.7.1 1. Sequential Communication

Agents pass results in order.

Agent A → Agent B → Agent C

22.7.2 2. Central Coordinator

One agent manages others.

Coordinator → Agents → Coordinator

22.7.3 3. Peer-to-Peer

Agents interact directly.

Agent A ↔ Agent B ↔ Agent C

The pattern depends on system complexity.

22.8 Coordination Strategies

22.8.1 1. Centralized Control

  • One agent orchestrates

  • Easier to manage

22.8.2 2. Decentralized Control

  • Agents decide independently

  • More flexible

22.8.3 Trade-off

  • Centralized → control

  • Decentralized → adaptability

22.9 Challenges in Multi-Agent Systems

22.9.1 1. Coordination Complexity

  • Managing interactions

  • Avoiding conflicts

22.9.2 2. Communication Overhead

  • More messages

  • Increased latency

22.9.3 3. Error Propagation

  • One agent’s mistake affects others

22.9.4 4. Debugging Difficulty

  • Harder to trace issues

More agents increase power, but also complexity.

22.10 Designing Multi-Agent Systems

To design effective systems:

22.10.1 1. Define Roles Clearly

  • Avoid ambiguity

22.10.2 2. Design Interfaces

  • Standardize communication

22.10.3 3. Control Interaction

  • Limit unnecessary communication

22.10.4 4. Add Evaluation Layers

  • Validate intermediate results

Structure is critical for coordination.

22.11 When to Use Multi-Agent Systems

22.11.1 Suitable Cases

  • Complex workflows

  • Multi-step reasoning

  • Systems requiring specialization

22.11.2 Not Suitable

  • Simple tasks

  • Low complexity

  • Single-step problems

Do not use multiple agents unless needed.

22.12 Multi-Agent Systems in the Minimal Loop

Multi-agent systems extend the execution phase:

Intent → Spec → Instruction → Execution (Multi-Agent) → Evaluation

Execution becomes:

  • A coordinated process

  • A network of interactions

22.13 Final Insight

Multi-agent systems represent a shift from:

  • Single-process execution

To:

  • Distributed intelligence

Systems are no longer single actors.
They are coordinated collectives.

22.14 Closing Statement

In the LLM era, complex systems are built not by increasing the power of one agent, but by organizing many.

The strength of a system lies in how well its agents collaborate.
