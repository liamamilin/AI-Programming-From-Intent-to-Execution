17  规格智能体（Spec Agent）

17.1 从手动定义到辅助定义

在基于模板的 Spec 中，人类直接定义规格说明，使用模板指导思维。这种方法在问题熟悉、结构清晰、用户理解系统时工作良好。

但在许多现实场景中，这些条件不满足：意图模糊、结构不清晰、用户不能轻松地将想法转化为 Spec。

这带来了一个挑战：当人类专业知识有限或问题被理解得较差时，我们如何创建好的 Spec？

这引领了一种新方法：

使用 AI 智能体生成规格说明

17.2 什么是规格智能体？（What is a Spec Agent?）

规格智能体（Spec Agent）是一个：

将人类意图转化为结构化规格说明的系统

它充当智能助手，帮助弥合模糊的人类想法与精确的机器可执行规格说明之间的差距。

规格智能体以自然语言描述、部分想法或片段、需要精化的不完整目标为输入，并产生一个结构化的 Spec，包含定义的输入、输出、约束和成功标准。

意图（非正式）→ 规格智能体 → Spec（结构化）

智能体不执行系统。它定义系统。

这是一个关键区别。规格智能体不是实现工具——它是定义工具。它帮助你在尝试构建之前澄清你想要什么。

17.3 为什么需要规格智能体（Why Spec Agents Are Needed）

规格智能体解决了几个关键挑战：

填补专业知识差距： 不是每个人都善于定义系统。规格智能体使没有经验的人能够创建高质量的 Spec。

加速定义过程： 手动创建 Spec 可能很慢。智能体可以在几秒钟内从粗略的想法生成结构化的 Spec。

减少遗忘错误： 人类经常忘记指定约束、边界情况或成功标准。规格智能体一致地包含所有必需的元素。

17.4 规格智能体的工作流（Spec Agent Workflow）

1. 接收意图： 智能体接收用户的自然语言描述。“我需要一个从文档中提取关键信息的系统。”

2. 交互式澄清： 如果意图不明确，智能体提出问题以澄清。关键信息涉及哪些文档？提取什么样的关键信息？输出格式是什么？

3. 生成 Spec： 智能体生成包含输入、输出、约束和成功标准的结构化 Spec。

4. 迭代精化： 用户可以审查 Spec 并提出修改要求，直到它满足需求。

17.5 人与智能体的合作（Human-Agent Collaboration）

规格智能体的目的不是取代人类判断。它是增强它。智能体处理结构化工作，人类做出关于什么是正确的最终决定。

有效的合作关系：智能体起草，人类审查和批准。智能体建议，人类决定。智能体提供完整的覆盖，人类提供上下文和理解。

规格智能体使规格创建更快、更一致，同时让人类保持最终的控制。
