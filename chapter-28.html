29  构建数据管道（Building a Data Pipeline）

29.1 数据管道作为系统

数据管道（data pipeline）是处理数据流的系统。它们从多个源接收输入，转换数据，并将结果存储在目标系统中。在传统软件中，管道是批处理或流处理作业。在 LLM 时代，数据管道通常涉及 AI 驱动的转换——文本清洗、分类、提取、总结、翻译。

数据管道的核心价值在于：它将原始、非结构化的数据转化为结构化、可操作的洞察。没有管道，AI 系统只能处理静态的、预定义的数据集。有了管道，AI 系统可以持续地从新数据中学习和适应。

29.2 定义意图（Define Intent）

意图： 构建一个管道，将来自多个渠道的原始客户反馈转化为结构化的可操作洞察。

精化后的目标： - 目标：每周自动生成客户反馈分析报告 - 对象：来自电子邮件、调查和聊天记录的原始反馈文本 - 上下文：每周处理 1000-5000 条反馈记录 - 约束：不存储个人身份信息（PII），分类准确率 > 85%，处理时间 < 2 小时/周 - 成功标准：每项反馈被正确分类，趋势被正确识别，报告在周一早上 9 点前可用

29.3 管道架构（Pipeline Architecture）

29.3.1 阶段 1：输入（Input）

来自多个源的原始文本。每个源有不同的格式和结构。

  • 电子邮件：非结构化文本，可能包含签名和转发链（需清洗）
  • 调查：结构化表格数据，字段名和类型
  • 聊天记录：对话记录，多轮文本，包含元数据

29.3.2 阶段 2：解析（Parse）

清理和标准化输入数据。

  • 文本清洗：移除签名、无关元数据、多余空白
  • 格式化：将不同源的输入转换为统一格式
  • 元数据提取：提取源、日期、类别标签

29.3.3 阶段 3：处理（Process）

AI 驱动的转换。

  • 分类（Classification）：将反馈分类为预定义类别（产品问题、服务问题、建议、表扬）
  • 情绪分析（Sentiment Analysis）：检测情绪倾向（正面、中性、负面）
  • 提取（Extraction）：提取关键短语、行动项、产品提及
  • 总结（Summarization）：对长反馈进行总结

29.3.4 阶段 4：聚合（Aggregate）

按期分组，计算统计信息，识别趋势。

  • 按类别聚合：各类别反馈的数量和占比
  • 趋势检测：关键指标随时间的变化
  • 异常检测：与基线相比的显著变化

29.3.5 阶段 5：输出（Output）

生成结构化报告。

  • 生成报告：格式化为标准报告模板（Markdown 或 HTML）
  • 分发：通过邮件或系统发送给利益相关者
  • 存档：持久化到数据库或文件系统供未来参考

29.4 Spec 示例：阶段 3 分类（Spec Example: Stage 3 Classification）

输入： 已解析的反馈文本（字符串，最多 2000 字符），元数据（源、日期、语言）。

输出： 分类标签列表，每类置信度分数（0-1）。

{
  "labels": [
    {"category": "产品问题", "confidence": 0.92},
    {"category": "服务问题", "confidence": 0.15}
  ]
}

约束： 每项反馈可以有多个标签，置信度 < 0.5 的标签被排除，使用预定义类别列表。

边界情况： 空文本返回空列表，多语言输入保持语言判断，非常规格式不崩溃。

29.5 管道实现模式（Pipeline Implementation Patterns）

1. 每个阶段是独立的模块，具有自己的 Spec：每个阶段有明确的输入和输出 Schema。可以独立测试和优化。替换单个阶段不影响其他阶段。

2. 阶段通过定义良好的接口通信：输入 Schema、输出 Schema 被显式定义。数据格式在阶段之间传递时保持一致。

3. 错误处理在每个阶段本地进行：一个阶段的错误不影响其他阶段。错误被记录并传递，而非直接导致管道崩溃。

4. 可观察性（Observability）：每个阶段记录其输入、输出和处理时间。日志提供管道的完整追踪。

29.6 评估管道质量（Evaluating Pipeline Quality）

逐阶段评估： 对每个阶段独立测试。输入样本 → 预期输出 → 实际输出 → 比较。

端到端评估： 使用完整的测试管道。从原始输入到最终输出。衡量整体准确率和延迟。

关键指标： - 分类准确率 > 85% - 情绪分析准确率 > 80% - 处理延迟 < 5 秒/条 - 管道成功率 > 99%

29.7 小结

数据管道是从原始数据到结构化洞察的桥梁。通过将管道分解为具有明确 Spec 的独立阶段，你构建了一个可测试、可维护、可演化的系统。

管道的设计原则——模块化、独立 Spec、错误隔离、可观察性——适用于任何 AI 数据管道。掌握了它们，你可以系统地构建从数据到价值的自动化工作流。
