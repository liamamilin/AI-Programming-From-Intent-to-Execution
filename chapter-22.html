23  Memory & Context

23.1 Why Memory Matters

So far, we have described systems that:

  • Receive input—accept user queries or data
  • Produce output—generate responses or results

These systems operate as if each interaction is isolated—each input is processed independently, with no memory of previous interactions.

However, many real-world systems require continuity:

  • Conversations span multiple turns—users expect the system to remember what was said
  • Decisions depend on past interactions—context from previous turns informs current decisions
  • Systems must adapt over time—learning from experience and evolving behavior

Without memory:

  • Each interaction is isolated—no connection between turns
  • Context is lost—previous information is unavailable
  • Behavior becomes shallow—system cannot maintain coherent conversation

Memory is what allows systems to persist beyond a single interaction, enabling continuity and depth.

23.2 What is Context?

Context is the information available to the system at the moment of execution. It represents what the system “knows” right now.

It includes:

  • Current input—the immediate query or task
  • Relevant past interactions—history that informs current processing
  • External knowledge—information retrieved from databases or other sources
  • System state—current status, configuration, or mode

23.2.1 Example

In a conversation:

  • Current message: “What was that about again?”
  • Previous messages: “I’m discussing AI memory systems.”
  • User preferences: “Prefers concise explanations”
  • Session data: “5 messages exchanged in this session”

All of these form the context for understanding and responding to the current message.

Context defines what the system “knows right now.” It determines how the system interprets and responds to input.

23.3 Context as a Working Set

Context can be understood as a working set of information—the subset of all available information that is active and relevant to the current task.

It is:

  • Limited in size—only so much information can be considered at once
  • Relevant to the current task—focuses on what’s needed right now
  • Dynamically constructed—changes based on task requirements

23.3.1 Characteristics

  • Temporary—exists only during execution
  • Task-specific—tailored to current needs
  • Continuously updated—changes as processing progresses

Context is not everything the system knows. It is what the system is using right now to perform the current task.

23.4 What is Memory?

Memory is the information stored beyond the current interaction. It represents:

What the system retains over time

Memory enables continuity—allowing the system to remember past interactions, learn from experience, and maintain coherence across sessions.

23.4.1 Types of Memory

23.4.1.1 1. Short-Term Memory

  • Recent interactions—last few turns of conversation
  • Session-based—persists only during current session
  • Volatile—lost when session ends

Uses: - Maintaining conversation flow - Tracking immediate context - Handling multi-turn dialogues

23.4.1.2 2. Long-Term Memory

  • Persistent storage—information kept across sessions
  • User data—preferences, history, patterns
  • Knowledge base—facts, relationships, and information

Uses: - Personalization—remembering user preferences - Learning—improving over time - Context retention—carrying information across sessions

Memory extends the system across time. Without memory, each interaction starts fresh, with no connection to the past.

23.5 Context vs Memory

It is important to distinguish between these two concepts, as they play different roles in system behavior:

23.5.1 Context

  • Active—currently being used for processing
  • Limited—bounded by capacity constraints
  • Immediate—directly available for execution

Purpose: Drive current decision-making and response generation

23.5.2 Memory

  • Stored—persisted for future use
  • Potentially large—can contain vast amounts of information
  • Persistent—survives across sessions and time

Purpose: Provide history, enable learning, support continuity

23.5.3 Relationship

Memory → Retrieval → Context → Execution

Memory feeds context through retrieval. Context drives execution. Together, they enable systems to operate with awareness of both the immediate situation and accumulated history.

Memory feeds context. Context drives execution. The relationship connects the past to the present.

23.6 Context Management

Context is limited by computational constraints—there’s only so much information that can be processed at once. Therefore, context must be managed carefully.

23.6.1 Challenges

  • Context window limits—models can only process a fixed amount of information
  • Irrelevant information—not all past interactions are useful now
  • Noise accumulation—too much information degrades performance

23.6.2 Strategies

23.6.2.1 1. Selection

  • Include only relevant information—filter what’s important
  • Prioritize recent interactions—newer information is often more relevant
  • Focus on task needs—select information that supports current objectives

23.6.2.2 2. Compression

  • Summarize past interactions—reduce long conversations to key points
  • Extract essential information—preserve meaning while reducing size
  • Create summaries—compress multiple turns into condensed representations

23.6.2.3 3. Prioritization

  • Keep important data—prioritize information that’s critical
  • Remove low-value data—discard information that’s not useful
  • Maintain balance—mix of recent and historically important information

Good context management improves system performance. It ensures the system focuses on what matters.

23.7 Long-Term Memory Systems

Long-term memory requires infrastructure and mechanisms to:

  • Store—persist information for future use
  • Retrieve—access stored information when needed
  • Update—modify existing information as needed

23.7.1 Storage

Long-term memory can be stored in various forms:

  • Databases—structured storage for organized data
  • Vector stores—semantic storage for similarity-based retrieval
  • Files—unstructured storage for documents and media
  • Graph databases—relationship-focused storage for connected information

Choice depends on: - Type of information - Access patterns - Query requirements

23.7.2 Retrieval

Retrieving information from long-term memory requires mechanisms to:

  • Search—find relevant information based on queries
  • Match—identify information that satisfies requirements
  • Rank—order results by relevance

Approaches: - Keyword search—match exact terms - Semantic search—match meaning and concepts - Hybrid—combine keyword and semantic approaches

23.7.3 Update

Memory systems must support:

  • Add new information—incorporate new data
  • Modify existing data—update or correct information
  • Remove outdated data—delete information that’s no longer relevant

Memory is only useful if it can be retrieved effectively. Storage without retrieval is useless.

23.8 Retrieval as a Core Mechanism

To use memory effectively, systems must retrieve relevant information at the right time. Retrieval is the bridge between memory and context.

23.8.1 Retrieval Flow

Query → Retrieve → Context → Generate
  1. Query—identify what information is needed
  2. Retrieve—search memory and fetch relevant information
  3. Context—incorporate retrieved information into current context
  4. Generate—use enriched context to produce response

23.8.2 Key Properties

  • Relevance—retrieved information must be related to current task
  • Efficiency—retrieval must be fast enough for real-time use
  • Accuracy—retrieved information must be correct and reliable

Retrieval connects memory to execution. It’s the mechanism that brings the past into the present.

23.9 Memory in Multi-Agent Systems

In multi-agent systems, memory can be organized in different ways:

23.9.1 Shared Memory

  • All agents access the same data
  • Centralized storage
  • Consistent across agents

Advantages: - Consistency—all agents have same information - Simplicity—single source of truth - - Coordination—easier to synchronize

Disadvantages: - Bottleneck—single point of access - - Scalability—may limit performance - - Complexity—harder to manage concurrent access

23.9.2 Distributed Memory

  • Each agent maintains its own memory
  • Decentralized storage
  • Independent management

Advantages: - Flexibility—agents can specialize - Scalability—no single bottleneck - - Performance—parallel access

Disadvantages: - Inconsistency—agents may have different information - - Complexity—harder to coordinate - - Redundancy—potential for duplication

23.9.3 Trade-offs

  • Shared → consistency, but limits flexibility
  • Distributed → flexibility, but harder to coordinate

23.10 Common Mistakes

Several common mistakes undermine effective memory systems:

23.10.1 1. Overloading Context

  • Too much information—exceeding capacity limits
  • Reduced performance—slower processing and degraded quality
  • Increased cost—more computation required

Solution: Select and compress. Include only what’s relevant.

23.10.2 2. Ignoring Memory

  • Systems forget important data—lose valuable information
  • No continuity—cannot maintain conversation
  • No learning—cannot improve over time

Solution: Design memory systems from the start. Make memory a first-class component.

23.10.3 3. Poor Retrieval

  • Irrelevant information—fetching data that’s not useful
  • Missed context—failing to retrieve important information
  • Slow performance—retrieval takes too long

Solution: Invest in retrieval mechanisms. Use semantic search, ranking, and relevance scoring.

23.10.4 4. Mixing Context and Memory

  • No clear separation—blurring the distinction
  • Hard to manage—difficult to optimize each component
  • Poor performance—both suffer from lack of focus

Solution: Maintain clear separation. Context is immediate working memory. Memory is persistent storage.

23.11 Designing Memory Systems

Effective memory systems require careful design:

23.11.1 Key Questions

  • What should be stored?—identify valuable information
  • When should it be retrieved?—determine retrieval triggers
  • How should it be updated?—define update mechanisms

23.11.2 Design Principles

  • Clear separation—distinguish context from memory
  • Efficient retrieval—optimize for speed and relevance
  • Controlled updates—manage how information changes over time

23.12 Memory in the Minimal Loop

Memory influences the execution phase of the Minimal Loop:

Intent → Spec → Instruction → Execution (with Context) → Evaluation

Context is constructed using memory before execution:

  1. Retrieve—fetch relevant information from memory
  2. Construct—build context from retrieved information and current input
  3. Execute—use enriched context to generate response

23.13 Final Insight

Memory and context determine how much of the past influences the present.

Without them:

  • Systems are reactive—responding only to immediate input
  • Behavior is shallow—no depth or continuity
    • Intelligence is limited—cannot build on experience

With them:

  • Systems become adaptive—learning and evolving
  • Behavior becomes coherent—maintaining consistency over time
    • Intelligence emerges—accumulating knowledge and wisdom

Intelligence emerges from the ability to use the past in the present. Memory and context enable this capability.

23.14 Closing Statement

In the LLM era, systems are no longer stateless processors. They are evolving entities that:

  • Remember—preserving information over time
  • Retrieve—accessing relevant information when needed
  • Adapt—learning and improving based on experience

Memory gives systems continuity. Context gives them focus.

23.15 Key Insight

Memory and context enable systems to maintain continuity across interactions. Key components include context as working set, short-term vs long-term memory, retrieval mechanisms, and memory in multi-agent systems (shared vs distributed).

23.16 Action Hint

Design memory systems from the start. Define what to store, when to retrieve, and how to update. Implement efficient retrieval with relevance scoring. Manage context carefully—select, compress, and prioritize. Distinguish clearly between context (immediate) and memory (persistent).
