24  构建智能体系统（Building an Agent System）

24.1 从概念到系统

在本节的前几章中，我们探讨了使 LLM 时代的系统构建成为可能的构成要素：工具使用、多智能体架构、记忆与上下文。现在是将它们组合在一起的时候了。

本章提供了一个结构化的过程，用于从概念到工作系统构建一个完整的智能体系统。

24.2 构建过程概览（Build Process Overview）

第 1 步：定义意图和 Spec 第 2 步：设计架构 第 3 步：实现核心组件 第 4 步：集成工具和记忆 第 5 步：测试和精化

24.3 第 1 步：定义意图和 Spec（Define Intent and Spec）

从清晰的意图开始。你想解决什么问题？

示例意图： “构建一个智能体，帮助开发者从错误消息中调试。”

精化后的 Spec： - 输入：错误消息、代码上下文（可选） - 输出：错误原因 + 解决方案建议 + 代码示例 - 约束：仅基于已知知识回答；在不确定时明确说明；支持常见编程语言 - 成功标准：针对已知错误，推荐的解决方案准确率 > 80%

24.4 第 2 步：设计架构（Design Architecture）

基于 Spec，设计系统架构：

主流程： 用户输入 → 解析错误 → 搜索知识库 → 匹配解决方案 → 格式化输出

模块： - 解析器：提取错误类型和消息 - 搜索器：在知识库中搜索匹配 - 求解器：生成解决方案建议 - 格式化器：格式化输出

数据流： 原始错误 → 结构化错误 → 匹配记录 → 解决方案 → 格式化响应

24.5 第 3 步：实现核心组件（Implement Core Components）

每个模块需要自己的 Spec 和指令。例如，解析器智能体：

解析器 Spec： - 输入：原始错误文本（字符串） - 输出：结构化错误（类型、消息、行号、语言） - 约束：识别常见错误模式；提取相关行号；确定编程语言

编译后的指令： “你是一个错误解析器。分析以下错误文本。提取错误类型（如 SyntaxError、TypeError）、错误消息、行号和编程语言。”

24.6 第 4 步：集成工具和记忆（Integrate Tools and Memory）

为系统增加能力：搜索数据库工具、代码执行沙箱、文档检索 API。

为系统增加记忆：记住常见的错误模式、缓存解决方案结果、从用户反馈中学习。

24.7 第 5 步：测试和精化（Test and Refine）

对照 Spec 进行系统测试：不同的错误类型、边界情况（不完整的错误消息、未知错误）、性能测试。

根据测试结果精化 Spec 和指令。

构建智能体系统是关于将 Spec 转化为工作系统——不是一次性完成，而是通过迭代精化。
